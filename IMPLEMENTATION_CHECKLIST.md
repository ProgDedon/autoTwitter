- [ ] Open http://localhost:3000 in browser
- [ ] Look for "Add Account" button (+ icon)
- [ ] Click "Add Account"
- [ ] Fill in the form:
  - [ ] Username: Your Twitter handle

  - [ ] Auth Type: Select "OAuth 1.0a"
  - [ ] Consumer Key: Paste from step above
  - [ ] Consumer Secret: Paste from step above
  - [ ] Access Token: Paste from step above

- [ ] Click "Connect"
- [ ] Wait for redirect to Twitter authorization page
- [ ] Click "Authorize" on Twitter

## Troubleshooting

### "Account not verified" error
- [ ] Make sure account has "verified: âœ…" next to it
- [ ] If not, you may have added it wrong
- [ ] [ ] Delete and re-add using "Add Account" button

### "No bearer token available" error
- [ ] Account doesn't have encryption credentials
- [ ] [ ] Use "Add Account" flow instead of manual entry

### "Unsupported Authentication" error (403)
- [ ] Token type is still wrong
- [ ] [ ] Delete account and completely re-add
- [ ] [ ] Ensure you're using OAuth 1.0a, not application-only token

### Server shows "Account not found"
- [ ] Account ID might be wrong
- [ ] [ ] Check `/api/accounts` endpoint to see correct IDs
- [ ] [ ] Restart server if accounts file was manually edited

### Button doesn't post anything
- [ ] Server might not be running
- [ ] [ ] Run `npm run server` in terminal
- [ ] [ ] Verify server outputs "listening on port 3001"

### No accounts appear in the list
- [ ] Server may not have loaded accounts.json
- [ ] [ ] Check server logs for errors
- [ ] [ ] Verify accounts.json exists in server/ folder
- [ ] [ ] Restart server: `npm run server`
